# Changelog
All notable changes to this project will be documented in this file.

## [1.0.0] - 2026-02-10

### Added
- **Reservation Feature**: Comprehensive room reservation system with full CRUD (Create, Read, Update, Delete) capabilities.
- **Room Information Feature**: Real-time room availability status based on selected dates.
- **Reservation History Feature**: Dedicated view for historical reservation data and archived bookings.
- **Search System**: Efficient server-side filtering for active reservations to improve data retrieval.
- **Security & Data Integrity**: Implemented DTOs (Data Transfer Objects) to secure API contracts and prevent over-posting.

## [1.1.0] - 2026-02-11

### Added
- **Hero Component**: Introduced a high-impact landing section featuring core branding 
- **About Component**: Added a minimalist feature grid highlighting key platform benefits: Real-time monitoring, Centralized History, and Data Transparency.
- **Footer Component**: Implemented a clean institutional footer with secondary navigation and copyright information.

### Changed
- **UI/UX Overhaul**: Reconstructed the interface from a "plain" state to a modern, minimalist aesthetic utilizing a solid Monochrome-Slate color palette.
- **Typography System**: Implemented a sharp typographic hierarchy with tracking-tighter headings and font-mono for technical IDs and room codes.
- **Table Architecture**: Refined all data displays (Room List, Reservations, and History) with slate-100 borders, hover-state highlights, and high-contrast pill-style status badges.
- **Enhanced SearchBar**: Improved the search experience with icon-prefixes, optimized internal padding, and a functional "Clear" (X) interaction.
- **Modal Refactoring**: Updated the View and Edit reservation modals with a backdrop-blur overlay and a structured grid layout for technical details.
- **Focused Form Layout**: Redesigned the reservation form into a centered "Card" layout to eliminate visual noise and improve user completion rates.
